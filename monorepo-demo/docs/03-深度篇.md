## CI/CD 自动化

### GitHub Actions 工作流

#### 基础 CI 流程

\`.github/workflows/ci.yml\`:

\`\`\`yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint-and-test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'pnpm'
      
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      
      - name: Lint
        run: pnpm lint
      
      - name: Type check
        run: pnpm type-check
      
      - name: Test
        run: pnpm test
      
      - name: Build
        run: pnpm build
\`\`\`

#### Turborepo 缓存集成

\`.github/workflows/ci-turbo.yml\`:

\`\`\`yaml
name: CI with Turbo

on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'pnpm'
      
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      
      - name: Cache Turbo
        uses: actions/cache@v3
        with:
          path: .turbo
          key: \${{ runner.os }}-turbo-\${{ github.sha }}
          restore-keys: |
            \${{ runner.os }}-turbo-
      
      - name: Build with Turbo
        run: pnpm turbo build --cache-dir=.turbo
      
      - name: Test with Turbo
        run: pnpm turbo test --cache-dir=.turbo
\`\`\`

#### Changesets 自动发布

\`.github/workflows/release.yml\`:

\`\`\`yaml
name: Release

on:
  push:
    branches:
      - main

jobs:
  release:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      
      - uses: actions/setup-node@v4
        with:
          node-version: 18
          cache: 'pnpm'
      
      - name: Install dependencies
        run: pnpm install --frozen-lockfile
      
      - name: Create Release Pull Request or Publish
        uses: changesets/action@v1
        with:
          publish: pnpm changeset publish
        env:
          GITHUB_TOKEN: \${{ secrets.GITHUB_TOKEN }}
          NPM_TOKEN: \${{ secrets.NPM_TOKEN }}
\`\`\`

### 按需部署

只部署变更的应用:

\`\`\`yaml
- name: Detect changes
  uses: dorny/paths-filter@v2
  id: changes
  with:
    filters: |
      web-app:
        - 'packages/apps/web-app/**'
      admin:
        - 'packages/apps/admin/**'

- name: Deploy Web App
  if: steps.changes.outputs.web-app == 'true'
  run: pnpm --filter @demo/web-app deploy:prod

- name: Deploy Admin
  if: steps.changes.outputs.admin == 'true'
  run: pnpm --filter @demo/admin deploy:prod
\`\`\`

### Tree-shaking 优化

#### 1. 使用具名导出

\`\`\`typescript
// ❌ 不利于 tree-shaking
export default {
  formatDate,
  formatMoney,
  formatPhone,
};

// ✅ 使用具名导出
export { formatDate, formatMoney, formatPhone };
\`\`\`

#### 2. 设置 sideEffects

\`\`\`json
{
  "name": "@demo/utils",
  "sideEffects": false  // 表示所有文件都无副作用,可以安全 tree-shake
}

// 或指定有副作用的文件
{
  "sideEffects": ["*.css", "src/polyfills.ts"]
}
\`\`\`

#### 3. 使用 ES Modules

\`\`\`json
{
  "type": "module",
  "exports": {
    ".": {
      "import": "./dist/index.js",
      "require": "./dist/index.cjs"
    }
  }
}
\`\`\`

### 代码分割策略

#### 1. 手动分割

\`\`\`typescript
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        manualChunks: {
          'vendor-react': ['react', 'react-dom', 'react-router-dom'],
          'vendor-ui': ['antd', '@ant-design/icons'],
          'vendor-utils': ['dayjs', 'lodash-es'],
        },
      },
    },
  },
});
\`\`\`

#### 2. 动态导入

\`\`\`typescript
// React 路由懒加载
const HomePage = lazy(() => import('./pages/HomePage'));
const UsersPage = lazy(() => import('./pages/UsersPage'));

// Vue 路由懒加载

  {
    path: '/',
    component: () => import('./views/Dashboard.vue'),
  },
];
\`\`\`

### 依赖去重

#### 查看重复依赖

\`\`\`bash

\`\`\`

\`\`\`json

    "overrides": {
      "lodash": "^4.17.21",
      "axios": "^1.6.7"
    }
  }
}

## 常见陷阱与解决方案

### 陷阱 1: 循环依赖

**问题**:

\`\`\`
@demo/utils → @demo/types → @demo/utils  (循环!)
\`\`\`

**解决方案**:

1. 重构代码,消除循环
2. 提取共同依赖到新包
3. 使用动态导入打破循环

### 陷阱 2: 版本不一致

**问题**: 不同包使用不同版本的 React

**解决方案**:

\`\`\`json
{
  "pnpm": {
    "overrides": {
      "react": "^18.2.0",
      "react-dom": "^18.2.0"
    }
  }
}
\`\`\`

### 陷阱 3: 构建顺序错误

**问题**: 应用构建时依赖的库还没构建

**解决方案**:

使用 Turborepo 管理构建顺序:

\`\`\`json
{
  "pipeline": {
    "build": {
      "dependsOn": ["^build"]  // 依赖包先构建
    }
  }
}
\`\`\`

### 陷阱 4: 幽灵依赖

**问题**: 代码使用了未声明的依赖

**解决方案**:

\`\`\`ini
# .npmrc
hoist=false
strict-peer-dependencies=true
\`\`\`

### 陷阱 5: 内存溢出

**问题**: 构建时内存不足

**解决方案**:

\`\`\`json
{
  "scripts": {
    "build": "NODE_OPTIONS='--max-old-space-size=4096' pnpm turbo build"
  }
}
\`\`\`

\`\`\`yaml

## 性能基准

### 构建速度对比

| 项目规模 | 无优化 | + Turborepo | + 远程缓存 |
|---------|--------|-------------|-----------|
| 小型(5包) | 30s | 15s | 2s |
| 中型(15包) | 120s | 45s | 5s |
| 大型(50包) | 600s | 150s | 10s |

### 缓存命中率

- **首次构建**: 0% 命中
- **无变更重建**: 100% 命中
- **修改 1 个包**: 80-90% 命中
