## Turborepo 构建优化

### 为什么需要 Turborepo?

随着包数量增加,构建时间会线性增长:

\`\`\`
无优化: 包A(30s) + 包B(20s) + 包C(15s) = 65s
使用 Turborepo: 
  - 并行构建: ~30s(最慢的包)
  - 缓存命中: ~0s
\`\`\`

### 核心概念

#### 1. 任务编排(Task Orchestration)

\`\`\`json
{
  "pipeline": {
    "build": {
      "dependsOn": ["^build"]  // ^ 表示依赖包先构建
    }
  }
}
\`\`\`

**依赖图示例**:

\`\`\`
web-app (应用)
  ↓ 依赖
ui-react (组件)
  ↓ 依赖
utils (工具)
  ↓ 依赖
types (类型)
\`\`\`

构建顺序: types → utils → ui-react → web-app

#### 2. 缓存(Caching)

Turborepo 会缓存任务输出:

\`\`\`bash
# 首次构建
pnpm turbo build
# ✓ @demo/types:build     5.2s
# ✓ @demo/utils:build     8.1s
# ✓ @demo/ui-react:build  12.3s
# ✓ @demo/web-app:build   15.7s
# 总计: 41.3s

# 再次构建(无变更)
pnpm turbo build
# ✓ @demo/types:build     CACHED
# ✓ @demo/utils:build     CACHED
# ✓ @demo/ui-react:build  CACHED
# ✓ @demo/web-app:build   CACHED
# 总计: 0.5s
\`\`\`

#### 3. 增量构建

只构建变更的包及其依赖者:

\`\`\`bash
# 只修改了 utils
pnpm turbo build

# 结果:
# ✓ @demo/types:build     CACHED (未变更)
# ✓ @demo/utils:build     8.2s (重新构建)
# ✓ @demo/ui-react:build  12.1s (依赖 utils,重新构建)
# ✓ @demo/web-app:build   15.5s (依赖 ui-react,重新构建)
\`\`\`

### 配置详解

#### turbo.json

\`\`\`json
{
  "$schema": "https://turbo.build/schema.json",
  "globalDependencies": ["**/.env.*local"],
  "pipeline": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**", "build/**"],
      "cache": true
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "lint": {
      "outputs": [],
      "cache": true
    },
    "test": {
      "dependsOn": ["^build"],
      "outputs": ["coverage/**"],
      "cache": true
    }
  }
}
\`\`\`

**配置说明**:

- `globalDependencies`: 全局文件变更会使所有缓存失效
- `dependsOn`: 任务依赖关系
- `outputs`: 缓存的输出目录
- `cache`: 是否启用缓存
- `persistent`: 任务是否持续运行(如 dev server)

### 使用技巧

#### 1. Filter 构建

\`\`\`bash
# 只构建 web-app 及其依赖
pnpm turbo build --filter=@demo/web-app...

# 只构建受 utils 影响的包
pnpm turbo build --filter=...@demo/utils

# 基于 git 差异构建
pnpm turbo build --filter=...[origin/main]
\`\`\`

#### 2. 查看依赖图

\`\`\`bash
# 生成依赖图
pnpm turbo build --graph
# 会生成 .turbo/graph.html
\`\`\`

#### 3. 跳过缓存(调试用)

\`\`\`bash
pnpm turbo build --force
\`\`\`

#### 4. 远程缓存

\`\`\`bash
# 链接 Vercel
pnpm turbo login
pnpm turbo link

# 之后团队成员可以共享缓存
\`\`\`

## Changesets 版本管理

### 版本管理工作流

#### 完整流程图

\`\`\`
开发 → 创建 Changeset → 提交 PR → 合并 → 发布
  ↓         ↓              ↓        ↓       ↓
代码修改  pnpm changeset  Review  Version  Publish
\`\`\`

### 实战场景

#### 场景 1: 修复 Bug

\`\`\`bash
# 1. 修复代码
# packages/shared/utils/src/format.ts
export const formatMoney = (amount: number) => {
  if (amount == null) return '-';  // 修复 bug
  // ...
};

# 2. 创建 changeset
pnpm changeset

┌─────────────────────────────────────┐
│ Which packages should be updated?   │
│ ✔ @demo/utils                       │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│ What kind of change?                │
│ ● patch (bug fix)                   │
│ ○ minor (new feature)               │
│ ○ major (breaking change)           │
└─────────────────────────────────────┘

请输入变更描述:
修复 formatMoney 处理 null 值的 bug

# 3. 生成文件
# .changeset/cool-cats-smile.md
---
"@demo/utils": patch
---

修复 formatMoney 处理 null 值的 bug

# 4. 提交
git add .
git commit -m "fix(utils): formatMoney null handling"
\`\`\`

#### 场景 2: 添加新功能

\`\`\`bash
# 1. 添加新功能
# packages/shared/ui-react/src/components/Modal.tsx
export const Modal: React.FC = () => { /* ... */ };

# 2. 创建 changeset
pnpm changeset

# 选择:
# - 包: @demo/ui-react
# - 类型: minor (新功能)
# - 描述: 添加 Modal 组件

# 3. 提交 PR
\`\`\`

#### 场景 3: 破坏性变更

\`\`\`bash
# 1. 重构 API
# packages/shared/types/src/user.ts
// 旧: interface User { name: string }
// 新: interface User { firstName: string; lastName: string }

# 2. 创建 changeset
pnpm changeset

# 选择:
# - 包: @demo/types
# - 类型: major (破坏性变更)
# - 描述: BREAKING CHANGE: 重构 User 类型,拆分 name 为 firstName 和 lastName
\`\`\`

### 发布流程

#### 准备发布

\`\`\`bash
# 1. 确保所有测试通过
pnpm test

# 2. 构建所有包
pnpm build

# 3. 更新版本
pnpm changeset version

# 这会:
# - 更新 package.json 版本号
# - 生成/更新 CHANGELOG.md
# - 删除已应用的 changeset 文件
# - 更新依赖版本

# 4. 提交版本变更
git add .
git commit -m "chore: 发布版本"

# 5. 发布到 npm
pnpm changeset publish

# 6. 推送 tag
git push --follow-tags
\`\`\`

#### CHANGELOG 示例

\`\`\`markdown
# @demo/utils

## 1.1.0

### Minor Changes

- a1b2c3d: 添加 formatCurrency 函数

### Patch Changes

- d4e5f6g: 修复 formatMoney 处理 null 值的 bug

## 1.0.0

### Major Changes

- Initial release
\`\`\`

## 跨框架代码共享

### 共享层级

\`\`\`
┌─────────────────────────────────────┐
│           应用层                     │
│  React App  │  Vue Admin  │  H5    │
├─────────────┴─────────────┴────────┤
│           UI 层                      │
│  @demo/ui-react  │  @demo/ui-vue   │
├──────────────────┴─────────────────┤
│           逻辑层                     │
│  @demo/utils  │  @demo/types       │
└─────────────────────────────────────┘
\`\`\`

### 共享策略

#### 1. 类型定义(完全共享)

\`\`\`typescript
// @demo/types
export interface User {
  id: string;
  name: string;
}

// React 中使用
import type { User } from '@demo/types';
const user: User = { id: '1', name: 'John' };

// Vue 中使用
import type { User } from '@demo/types';
const user: User = { id: '1', name: 'John' };
\`\`\`

#### 2. 工具函数(完全共享)

\`\`\`typescript
// @demo/utils
export const formatDate = (date: Date) => {
  return dayjs(date).format('YYYY-MM-DD');
};

// React 和 Vue 都可以使用
import { formatDate } from '@demo/utils';
\`\`\`

#### 3. UI 组件(分别实现)

\`\`\`typescript
// @demo/ui-react
export const Button: React.FC = (props) => { /* React 实现 */ };

// @demo/ui-vue
export const Button: DefineComponent = defineComponent({ /* Vue 实现 */ });
\`\`\`

### 注意事项

1. **避免框架特定代码进入共享层**

\`\`\`typescript
// ❌ 不要在 utils 中使用 React Hooks
export const useFormatDate = () => {
  return useCallback((date) => { /* ... */ }, []);
};

// ✅ 保持纯函数
export const formatDate = (date: Date) => { /* ... */ };
\`\`\`

2. **使用条件导出**

\`\`\`json
{
  "exports": {
    ".": "./dist/index.js",
    "./react": "./dist/react.js",
    "./vue": "./dist/vue.js"
  }
}
\`\`\`

## 实战案例

### 案例 1: 添加新的工具函数

**需求**: 在 `@demo/utils` 中添加 `formatCurrency` 函数

\`\`\`bash
# 1. 实现功能
# packages/shared/utils/src/format.ts
export const formatCurrency = (amount: number, currency = 'CNY') => {
  return new Intl.NumberFormat('zh-CN', {
    style: 'currency',
    currency,
  }).format(amount);
};

# 2. 导出
# packages/shared/utils/src/index.ts
export { formatCurrency } from './format';

# 3. 构建
pnpm --filter @demo/utils build

# 4. 在应用中使用
# packages/apps/web-app/src/pages/HomePage.tsx
import { formatCurrency } from '@demo/utils';
console.log(formatCurrency(1234.56)); // ¥1,234.56

# 5. 创建 changeset
pnpm changeset
# 选择: @demo/utils, minor, "添加 formatCurrency 函数"

# 6. 提交
git add .
git commit -m "feat(utils): add formatCurrency"
\`\`\`

### 案例 2: 跨包重构

**需求**: 将 `User` 类型从 `@demo/utils` 移到 `@demo/types`

\`\`\`bash
# 1. 在 types 中添加
# packages/shared/types/src/user.ts
export interface User { /* ... */ }

# 2. 从 utils 中删除并重新导出
# packages/shared/utils/src/index.ts
export type { User } from '@demo/types';

# 3. 构建两个包
pnpm --filter @demo/types build
pnpm --filter @demo/utils build

# 4. 测试所有应用
pnpm test

# 5. 创建 changesets
pnpm changeset
# @demo/types: minor (添加 User 类型)
pnpm changeset
# @demo/utils: patch (重新导出 User 类型)

# 6. 提交
git add .
git commit -m "refactor: move User type to @demo/types"
\`\`\`
